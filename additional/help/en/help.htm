ClipKA

This program is a multiple clipboard and hotkey manager. Replacing incorrect encoding, replacing incorrectly typed text case, tasks in Groovy script language.

The data is stored in sqlite files, located in the dat folder. Files can be deleted, will be reinitialized at startup.

Command prompt:
-p [Port], --port [Port] - change socket port
-t [Number], --task [Number] - run task with id = Number
-g [Number], --group [Number] - show tasksgroup with id = Number

Limitations:

Version Java: 11

Minimum version of Windows: Windows Vista

To check the launch of the application, the port on the computer opens. To run multiple applications, you need to use different ports.

Database with clipboard entries is not protected. You can restrict the record to the database by filtering window headers/classes.

Settings

System settings

"Socket port". At the first start, the program reserves the specified port.When you restart, the port is checked and prevents the second copy of the program from running. Another use is to run the task from the command line.

"Show window with action buttons" - The program has a regular window with buttons of main actions. When the check box is selected, the window will be displayed.

"Show in system tray". If the check box is not selected, a window with action buttons is displayed.

"Show notification after task with error". If the check box is not selected, windows with notifications is not displayed. However, you can view the records in the 'Alerts and Errors'

"Default timeout for notification (sec)". If the value is not set, windows with notifications is not closed.

Clipboard settings

"Number of items on page".

"Maximum text size"

"Remove duplicates items". If you delete duplicates, you may lose information about the time of creation and source of the clips.

"Timeout for save position (sec)". To easily select several consecutive clips, the previous position is saved when the list is opened. Later specified time - the list of clips opens, starting with the most recent.

"Separator of clip texts when selecting multiple clips in the clip list". If you select multiple clips in the list, you can get them in one string. Each clip will be separated from the other by the specified text.

"Watch the clipboard: primary (mouse selection)". Only Linux.

Clips list

A list of clips showing the copy time, title, and class of the owner window is displayed. For Linux system additionally shown clipboard: normal or selection. When you select a clip, the current position is stored. When you reopen the list, the cursor is placed on the previous clip. The cursor position is reset after the time has elapsed, specified in the settings.

Tasks groups

Tasks can be grouped together. When you open a task group, you can view/run tasks only from the specified group. You can specify hotkeys for a group. When you enter these keys, a list of tasks opens with selection for the selected group.

Tasks

You can perform actions on rows retrieved from the clipboard or selected in a text editor. Each task consists of a list subtasks executed sequentially.

Data exchange between subtasks is carried out using a special a dictionary repository where you can place text content by specifying any code and also get back. If the dictionary code is empty -- we are working with the current row being processed.

It should be taken into account that there are two special subtasks: copy to the clipboard and paste from the clipboard. Despite the indication keyboard shortcuts, these subtasks are special. In some cases (e.g. when working with an RDP server) updating the clipboard may take for a long time and you cannot use just simulating keystrokes. You must wait for the actual buffer update.

Filter for foreground window

The task can only be performed in the specified window or screen item. You can specify in regular expression format a allowed or denied character combination in a window header or class. You can, for example, use a filter to avoid copying saved clips to the list passwords.

Alerts and errors

This is a separate table that automatically gets errors when performing tasks. In addition, you can write text there yourself so that then analyze.
